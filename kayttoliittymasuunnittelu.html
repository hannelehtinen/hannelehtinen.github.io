Mockup ja Prototyyppi

Käyttöliittymäsuunnittelusta minulla ei ole varsinaista aikaisempaa kokemusta. Projektin alkaessa otin käyttööni Adobe XD -ohjelman, joka ei ollut entuudestaan tuttu. Suoritin ohjelmaa varten Udemyn Adobe XD Basics Course -kurssin ja sen avulla sain luotua toiminnallisen prototyypin valmiista tuotteesta. Adobe XD:n lisäksi tein mockup-luonnoksia myös Procreatella, Affinity Designerilla sekä käsin.

Prototyyppi

käyttöliittymäsuunnittelu


Valmis prototyyppi on rakenteeltaan melko samankaltainen kuin varsinainen tuote. Eroavaisuuksia prototyypissä on yksittäiset visuaaliset elementit, kuten ikonit, taustakuva ja varsinaiset tutoriaalinappulat. Lopullisen käyttöliittymän toteutin pääasiassa prototyyppiä mukaillen. Sovelluksen ulkoasu on toteutettu CSS:llä ja jonkin verran Angular Material -elementeillä, jotka olen itse muokannut sovelluksen tyyliin sopivaksi. Olin pääasiallisesti vastuussa sovelluksen visuaalisesta ilmeestä ja tyylittelystä joka komponentissa.



Tutoriaali


Alla tutoriaalin suoritustoiminto, johon olen tehnyt ulkoasun. Tutoriaali on kokonaisuudessaan responsiivinen.



Alla tutoriaalin aloitusikkunan animaation HTML-koodi.




Alla HTML-koodi, kun harjoite on suoritettu ja käyttäjän taso nousee.



Alla CSS-koodi maskoteille eri tilanteissa. Eri elementit, kuten asusteet ja tassu on rakennettu erillisinä kuvina, jotka asetellaan päällekkäin. Päälle, tassulle ja päässä olevalle hatulle on määritetty sama wobble-animaatio.

  .body, .face, .accessory, .book, .paws, .thumb, .thumbsUp, .thumbsUpAccessory {
      position: absolute;
      width: 200px;
      height: 200px;
  }
  .face, .thumb, .accessory {
      z-index: +1;
      animation: wobble 1s ease-in-out infinite;
      animation-direction: alternate;
  }
  
    @keyframes wobble {
      from {
          transform: rotateZ(0deg);
      }
      to {
          transform: rotateZ(3deg);
      }
    }


Kauppa ja Inventaario


Kauppa ja Inventaario -ikkuna on rakennettu CSS Grid:in avulla. Grid oli minulle ennestään pääpiirteittäin tuttu, mutta en kokemusta siitä ei pahemmin ollut. Responsiivisuus



Sovellus on suunniteltu alunperin mobiilille ja responsiivisuuden olen lisännyt jälkeenpäin. Olennaisin muutos nettisivuversiossa on alapalkin siirtyminen yläpalkiksi, sekä erillinen ikkuna, jossa näkyy käyttäjän tärkeimmät tiedot.



Alla HTML-koodi Stats-ikkunasta. Koska ikkuna on tarpeellinen vain Kauppa ja etusivulla, on se lisätty molempien sivujen komponentteihin erikseen. Toteutin ikkunan aluksi Yläpalkin komponenttiin, mutta se tuli näkyviin silloin joka ikkunassa, mikä ei ollut tarpeellista.



Stats-ikkunan päämuotoilu ja responsiivisuus.
